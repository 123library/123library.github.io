任意文件包含漏洞

2024-01-26 | Hits | 552 Words | 2 Minutes

文件包含（不是漏洞 是功能）

相同代码重复出现在不同文件 冗余 ->文件包含 更高效

函数

include() –临时加载引用 包含文件不存在 仍会往下执行

require() –在运行php之前读取文件一起执行 包含文件不存在,停止

只包含一次：

include_once()

require_once()

1.php

<?php
echo 'a';
?>

2.php

<?php
echo 'aaaaaaaaaaaaa<br/>';

include('1.php');
echo ' aaaaaaaaaaaaa ';
?>

任意文件包含(漏洞) :可以通过目标包含**文件

RFI/LFI ->远程文件包含 本地文件包含

R –remote L–localhost

本地文件包含:要包含的文件必须在目标主机上

远程文件包含：无论文件在哪里只要能访问到

​ php默认不开启 [开启 allow_url_include=On]

​ *php伪协议 php://input

危害：

任意文件都可被当作php执行

小tips

1.路径判断

sadhjb/../ 当前目录

cmd中的路径可包含特殊字符(除&和中文字符 &表示几条命令同时执行)

php中路径不可包含特殊字符

2.XX::xxxx php调用类里的方法

3.GET传参原本会url解码一次

PHPMYADMIN

高权限:日志写shell法

​ 1.路径 不知道网站根目录没办法

​ 2.没有root权限 没法设置

general_log 开启

general_log_file

1.查看general log是否开启

show variables like ‘%general%’;

2.打开general_log日志读写功能

SET global general_log=‘on’;

3.写shell，会把执行的语句写到shell.php，然后我们查询一下

SET GLOBAL general_log_file=‘C://phpStudy//PHPTutorial//WWW//shell.php’;
